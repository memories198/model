测试分类
在 Go 中，编写的测试用例可以分为四类：

单元测试：测试函数名称以 Test 开头，如 TestXxx、Test_Xxx，用来对程序的最小单元进行测试，如函数、方法等。

基准测试：也叫性能测试，测试函数名称以 Benchmark 开头，用来测量程序的性能指标。

示例测试：测试函数名称以 Example 开头，可以用来测试程序的标准输出内容。

模糊测试：也叫随机测试，测试函数名称以 Fuzz 开头，是一种基于随机输入的自动化测试技术，适合用来测试处理用户输入的代码，在 Go 1.18 中被引入。


在Go语言中，变量逃逸到堆上通常是由于变量的生命周期超出了其原始的作用域，或者由于闭包捕获了外部变量。以下是一些常见情况下变量会发生逃逸的情况：
    1. 变量在函数返回后仍被引用
        如果一个变量在函数返回后仍然被引用，Go编译器会将该变量分配到堆上，而不是栈上。
            func createPointer() *int {
                x := 42
                return &x
            }
        在这个例子中，变量 x 被返回为一个指针，因此它需要在函数返回后继续存在，编译器会将其分配到堆上。

    2. 接口类型的使用
        当使用接口类型时，如果具体类型的值需要在接口中存储，编译器可能会将其分配到堆上。
            func doSomething(i interface{}) {
                fmt.Println(i)
            }
        在这个例子中，传递给 doSomething 函数的具体类型的值可能会逃逸到堆上。

    3. 字符串和切片的使用
        字符串和切片的底层数组如果被赋值给长生命周期的变量或者被返回，也会导致逃逸。
            func createSlice() []int {
                s := make([]int, 10)
                return s
            }
        在这个例子中，底层数组 s 可能会被分配到堆上，因为它在函数返回后需要继续存在。
